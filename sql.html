Databases

What are databases?

A database is a collection of data organized in tables, stored and accessed by computer systems.

Each table contains rows and columns, which are know as records and fields respectively.


Functionality

Databases all have the following basic functionality - create, read, update, delete.

Each operation on a database (transaction) must satisfy ACID criteria. This means that each transaction must succeed or fail as a complete unit, without partially completing an operation. ACID stands for:

  1. Atomicity - a change to a database is either completely performed, or not at all. A half completed change must not be saved to the database.
  2. Consistency - any change in the database must retain the overall ] state of a database. For example, money subtracted from one account must be added to another account, or the money will literally dissappear.
  3. Isolation - a transaction must not be able to unterrupted by another transaction, preventing race conditions.
  4. Durability - once a change has been made to a database, it must not be lost due to a system failure.


SQL

What is SQL?

SQL (aka Structure Query Language) is a programming language used for managing data held in a relational database. It is used by software or clients to communicate with the DBMS.


Reading Data

The fundamental to reading data in SQL is the SELECT keyword. All SQL read queries are structured as
"SELECT fields FROM table WHERE condition".

For example, selecting the name and address of all students who are older than 15 could be done with
"SELECT name, address FROM students WHERE age >= 15".


Deleting Data

Similar to reading, deleting records in SQL also uses a keyword. All SQL delete queries are structured as
"DELETE FROM table WHERE condition".

For example, deleting all students in year 13 could be done with the command
"DELETE FROM students WHERE yr == 13".


Updating Data

Data doesn't just have to be deleted and remade to change a value. You can change individual fields using the
"UPDATE table SET field=value WHERE condition"
structure.

For example, increasing the year field for every student that passed the final exam could be done with
"UPDATE students SET yr=yr+1 WHERE finalExamPassed"


Creating Records

Creating new records can be done with the insert command. It is used in the format:
"INSERT INTO table VALUES (value1, value2, value3)
structure.

For example, inserting my data into a table with fields (id, name, age, year) could be done with
"INSERT INTO students VALUES (3209, 'Miles Watson', 16, 12)"


Deleting Tables

Tables can be deleted using the DROP keyword in the structure
"DROP TABLE table"
They can also be cleared using the TRUNCATE keyword:
"TRUNCATE TABLE table"

Boolean Logic

Boolean logic can be used to take into account multiple conditions. The keywords AND and OR behave like && and || in most programming languages - AND returns TRUE if both A and B are TRUE, OR returns TRUE if either A or B are TRUE.

Wildcards

There are a few wildcards that can be used in SQL. The most famous of which is the * wildcard, used to return all fields of a table.
"SELECT * FROM table WHERE condition"
You can also used wildcards to match strings. For example, you can find all the people that have the letter "A" before the letter "E" in their name with the following statement:
"SELECT name FROM students WHERE name LIKE "%a%e%"

Joining Tables

Getting data from multiple tables can be done two ways - with a JOIN statement or with nested SELECTS. Say you have two tables, students with fields (id, name) and grades with fields (studentID, subject, grade). If I wanted to get the Physics grade of "Miles Watson", I could do
"SELECT grade FROM students JOIN grades ON (id=studentID) WHERE name='Miles Watson'"
